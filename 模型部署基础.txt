# 模型部署基础自证

## 1. 模型小型化

**低比特量化**：掌握将浮点权重/激活值映射到低比特整数（INT8/INT4）的量化方法；理解训练后量化（PTQ, 无需重训练，精度损失较大）与量化感知训练（QAT, 在训练中模拟量化噪声，精度损失小）的区别；熟悉对称量化与非对称量化的设计差异——非对称量化通过 zero point 偏移适配非对称分布（如 ReLU 输出）。

**混合精度量化**：掌握对网络不同层使用不同量化比特数的策略——敏感层（如第一层/最后一层）保持高精度，中间层使用低精度；理解通过敏感度分析（逐层量化后测量精度下降）确定各层最优比特分配的方法；熟悉 GPTQ（逐层校准，基于 Hessian 二阶信息最小化量化误差）和 AWQ（基于激活值分布识别重要通道保护）在大模型量化中的应用。

**结构化剪枝**：掌握按结构化粒度（filter/channel/layer 级别）移除冗余参数的方法；理解结构化剪枝相比非结构化剪枝（单个权重置零）的工程优势——可直接获得硬件加速而无需稀疏计算支持；熟悉基于范数（L1 norm 最小的 filter 优先剪除）和基于梯度（对损失影响最小的通道优先剪除）的重要性评估方法。

**知识蒸馏**：掌握 teacher-student 框架——大模型（teacher）的 soft label（softmax 输出的概率分布，含类间相似性信息）指导小模型（student）学习；理解温度系数 T 的作用——T 越大 softmax 分布越平滑，暴露更多类间关系的 dark knowledge；熟悉 feature-based 蒸馏（中间层特征对齐）和 relation-based 蒸馏（样本间关系对齐）的扩展方法。

**AutoML**：掌握神经网络架构搜索（NAS）的核心思想——在预定义的搜索空间中自动寻找最优网络架构；理解搜索策略从 RL-based（NASNet, 用 controller 采样架构评估性能）→ 权重共享（ENAS, One-Shot 超网训练后搜索）→ 可微分搜索（DARTS, 连续松弛离散选择实现梯度优化）的效率演进；熟悉 NAS 在端侧高效模型设计（MobileNetV3、EfficientNet）中的实际应用。

**结构重参数化**：掌握训练时使用多分支复杂结构（如 RepVGG 训练时 3×3 conv + 1×1 conv + identity 三分支），推理时通过数学等价合并为单一卷积的技术；理解重参数化的核心优势——训练时多分支提升表征能力，推理时单分支保证速度和硬件友好；熟悉 BN 折叠（将 Batch Normalization 的 scale/shift 融合进卷积权重）作为基础重参数化操作。

## 2. 算子加速算法

**im2col**：掌握将卷积操作转化为矩阵乘法（GEMM）的经典方法——将输入特征图的每个卷积窗口展开为列向量组成矩阵，与卷积核矩阵相乘；理解 im2col 的优势在于充分利用高度优化的 BLAS 矩阵乘法库，缺点是内存占用增大（数据冗余展开）。

**Winograd**：掌握 Winograd 快速卷积通过减少乘法次数加速小卷积核（3×3）计算的原理——将卷积转化为 element-wise 乘法，以额外加法换取更少乘法（乘法远比加法耗时）；理解 Winograd 变换公式 Y = A^T[(GgG^T) ⊙ (B^TdB)]A 中各变换矩阵的作用；熟悉 Winograd 在 3×3 卷积加速中广泛应用（ARM CPU 加速的核心算法之一）以及数值精度损失的局限。

**FFT 快速卷积**：掌握利用卷积定理——时域卷积等价于频域乘法——将卷积转化为 FFT→频域逐元素相乘→IFFT 的计算流程；理解 FFT 卷积在大卷积核场景下的计算复杂度优势 O(n log n) vs O(n²)，但在小卷积核（3×3）场景下因 FFT 变换开销反而不如 Winograd/im2col。

## 3. 模型部署平台

### 端侧部署平台

**DLLite**：熟悉华为端侧轻量级推理框架，支持 HiAI Foundation 模型在华为设备上的高效推理。

**MindSpore Lite**：掌握华为全场景 AI 框架的端侧推理引擎；熟悉模型转换流程（MindSpore/ONNX/TF 模型→MindSpore Lite .ms 格式）；理解算子融合、量化压缩等端侧优化策略。

**TensorFlow Lite**：掌握 Google 端侧推理框架的核心流程——模型转换（TFLite Converter: FlatBuffers 格式）→ 量化（动态量化/全整数量化）→ 部署（Interpreter 加载执行）；熟悉 TFLite Delegate 机制将计算委托到 GPU/NPU 加速。

**MNN**：熟悉阿里端侧推理框架，支持多后端（CPU/GPU/NPU）调度；理解其轻量化设计（极小二进制体积）和动态图推理能力。

**QNN**：掌握 Qualcomm 神经网络推理引擎在骁龙平台的部署——支持 Hexagon DSP/GPU/CPU 多后端，针对高通硬件深度优化；熟悉模型转换和量化校准流程。

**NNIE**：熟悉海思 NPU 推理引擎（Neural Network Inference Engine），支持海思芯片上 CNN 模型的硬件加速推理；了解 NNIE 支持的算子范围和自定义算子开发流程。

### 云侧部署平台

**MLOps (MTP/MEP)**：掌握模型训练平台（MTP）和模型推理平台（MEP）的全流程——数据准备→模型训练→模型评估→模型发布→在线推理→监控告警；理解 MLOps 的核心价值在于将 ML 实验工程化，实现模型的可重复、可追踪、可迭代的全生命周期管理；具备在 WiseMLOps 平台完成模型部署并为上游提供调用服务的实践经验。

**MindSpore**：掌握华为全场景深度学习框架的核心特性——自动微分、自动并行（数据并行/模型并行/流水线并行自动切分）、图算融合优化；理解其在昇腾 NPU 上的原生优化优势。

**TensorRT**：掌握 NVIDIA GPU 推理优化引擎的核心技术——层融合（将 Conv+BN+ReLU 合并为单一 kernel）、精度校准（FP32→FP16/INT8 自动校准）、动态 tensor 内存复用、kernel 自动调优（从候选实现中选择最优 kernel）；理解 TensorRT 将模型编译为高度优化的引擎文件的部署流程。

**TensorFlow Serving**：掌握 Google 模型服务化框架——gRPC/REST API 提供推理服务、模型版本管理（自动加载最新版本、AB 测试灰度发布）、批处理调度（dynamic batching 提升吞吐）；理解其在生产环境高可用部署中的工程实践。

## 4. 我的学习感悟

模型部署这块是我从"做实验"到"做工程"转变最大的一段经历。

在学校和做研究的时候，我几乎不关心模型的推理速度和内存占用，跑得出结果就行。第一次接触线上部署才发现，latency 50ms 和 200ms 的差距在用户体验上是天壤之别，一个能用一个不能用。

量化是我最早接触的优化手段。第一次做 INT8 量化的时候，我直接用 PTQ 把整个模型量化了一遍，精度掉了三个点，当时觉得不可接受。后来学到混合精度量化的思路——敏感层保持高精度、不敏感层用低精度——同样是 INT8，精度只掉了零点几个点，推理速度快了将近一倍。这让我理解到，量化不是一刀切的事情，需要对模型各层的特性有细致的了解。

知识蒸馏是另一个让我印象深刻的技术。我做过一次把 BERT-large 蒸馏到一个 4 层小模型的实验，最终模型大小缩了 10 倍、速度快了 8 倍，但效果只掉了不到 2%。当时觉得 soft label 里的 dark knowledge 真的很神奇——teacher 模型在错误类别上的概率分布其实包含了大量关于类间关系的信息，这不是 hard label 能传递的。

TensorRT 是我在 GPU 部署上用得最多的工具。说实话第一次用的时候被各种报错折腾得很痛苦——动态 shape 支持有坑、某些算子不兼容需要写 plugin，但一旦调通，加速效果确实惊人。FP16 模式下基本可以做到精度无损、速度翻倍。

最大的一个教训是关于端侧部署的。有一次我把一个模型部署到手机端，在开发板上测试一切正常，但到了真实设备上发现发热严重、掉帧厉害。后来才搞明白是因为 NPU 算子覆盖不全，很多 op 回退到了 CPU 执行。这件事让我养成了一个习惯：部署前一定要先确认目标平台的算子支持列表，该做算子替换或图优化的提前做好。
